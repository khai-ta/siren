# SIREN API Reference

Complete API for detection, investigation, retrieval, and feedback.

## Detection Layer

### `detection.detect()`

Main entry point: runs statistical + ML anomaly detection, deduplicates, classifies incident type.

```python
from detection import detect

anomalies, incident = detect(
    rows: List[Dict[str, Any]],
    z_threshold: float = 3.0,
    contamination: float = 0.05,
) -> Tuple[List[Dict[str, Any]], Dict[str, Any]]
```

**Parameters:**
- `rows` — List[Dict] with keys: timestamp, service, error_rate, latency_p99, latency_p50, rps, cpu, memory (all except timestamp/service should be numeric)
- `z_threshold` — Z-score threshold for statistical detector (default 3.0, must be > 0)
- `contamination` — Anomaly contamination ratio for IsolationForest (default 0.05, range 0.01-0.5)

**Returns:**
- `anomalies` — List[Dict] deduplicated within 5-minute windows
  ```python
  {
      "timestamp": "2026-04-21T06:27:15+00:00",  # ISO 8601
      "service": "database",
      "metric": "error_rate",                    # One of the 6 metrics
      "value": 0.0105,                           # Actual metric value
      "zscore": 643.1,                           # Z-score magnitude
      "baseline_mean": 0.001,
      "baseline_std": 0.000015,
      "detector": "statistical",                 # or "isolation_forest"
      "changepoint": False,                      # True if PELT detected shift here
  }
  ```
- `incident` — Dict with classification and severity
  ```python
  {
      "incident_id": "compute_20260421T062715",  # Derived from first anomaly
      "timestamp": "2026-04-21T06:27:15+00:00",  # First anomaly timestamp
      "affected_services": ["database", "api-gateway"],
      "anomaly_type": "compute",                 # compute|network|database|memory|timeout
      "severity": "critical",                    # critical|high|medium
      "triggering_metrics": ["error_rate", "latency_p99"],  # Metrics that triggered classification
  }
  ```

**Example:**
```python
import csv
from detection import detect

# Load metrics from CSV
with open("metrics.csv") as f:
    metrics = list(csv.DictReader(f))
    for row in metrics:
        for key in ["error_rate", "latency_p99", "rps", "cpu", "memory"]:
            row[key] = float(row[key])

# Detect anomalies
anomalies, incident = detect(metrics)

print(f"Incident type: {incident['anomaly_type']}")
print(f"Affected: {', '.join(incident['affected_services'])}")
print(f"Anomalies: {len(anomalies)}")
```

### `detection.statistical.detect_statistical()`

Statistical detection only (z-score + PELT changepoint).

```python
from detection.statistical import detect_statistical

    rows: List[Dict],
    z_threshold: float = 3.0,

```

### `detection.ml_detector.detect_isolation_forest()`

ML detection only (IsolationForest).

```python
from detection.ml_detector import detect_isolation_forest

anomalies = detect_isolation_forest(
    rows: List[Dict],
    contamination: float = 0.05,

```

### `detection.trigger.classify_incident_type()`

Classify incident type from anomalies.

```python
from detection.trigger import classify_incident_type

incident_type = classify_incident_type(anomalies: List[Dict]) -> str
```

Returns: `"compute"`, `"network"`, `"database"`, `"memory"`, `"timeout"`, or `"compute"` (fallback)

## Investigation Layer

### `agent.run.run_investigation()`

LangGraph agent: Plan → Investigate → Verify → Report

```python
from agent.run import run_investigation

state = run_investigation(
    anomalies: list[dict],
    origin_service: str,
    window_start: str,
    window_end: str,
    incident_id: str,
    incident_type: str = "compute",
    max_steps: int = 15,
) -> dict[str, Any]
```

**Parameters:**
- `anomalies` — List of anomaly dicts from detection layer
- `origin_service` — Primary affected service (string)
- `window_start` — ISO timestamp string (5 min before first anomaly)
- `window_end` — ISO timestamp string (5 min after last anomaly)
- `incident_id` — Unique identifier (from incident classification)
- `incident_type` — Classified type (compute|network|database|memory|timeout); used to apply retrieval weights
- `max_steps` — Max tool calls (default 15, range 1-30); higher = more thorough but slower

**Returns:** InvestigationState dict with all fields populated:
```python
{
    "incident_id": "compute_20260421T062715",
    "anomalies": [...],                  # Input anomalies
    "origin_service": "database",        # Input origin service
    "incident_type": "compute",          # Input incident type
    "investigation_plan": ["Step 1", ...],  # Generated by Planner
    "current_step": 7,                   # Steps completed
    "hypotheses": [                      # Evolved during investigation
        {
            "id": "hyp_001",
            "statement": "database connection pool exhaustion",
            "confidence": 0.85,
            "evidence_for": ["ev_001", "ev_003"],
            "evidence_against": [],
            "status": "open" | "closed"
        }
    ],
    "tool_history": [
        {
            "step": 1,
            "tool_name": "query_logs",
            "arguments": {...},
            "result_summary": "Found 12 timeout errors",
            "timestamp": "2026-04-21T06:28:15"
        },
        ...
    ],
    "evidence_ledger": {
        "ev_001": {"tool": "query_logs", "summary": "..."},
        "ev_002": {"tool": "get_metrics", "summary": "..."},
    },
    "final_root_cause": "database_connection_pool_exhaustion",
    "final_confidence": 0.87,           # 0.0-1.0
    "final_report": "# RCA Report\n\n..."  # Markdown
}
```

**Example:**
```python
from investigation import run_investigation as investigate
from detection import detect
import csv

# Load metrics
with open("metrics.csv") as f:
    metrics = list(csv.DictReader(f))
    
# Detect
anomalies, incident = detect(metrics)

# Investigate
result = investigate(
    anomalies=anomalies,
    origin_service=incident["affected_services"][0],
    window_start=anomalies[0]["timestamp"],
    window_end=anomalies[-1]["timestamp"],
    incident_type=incident["anomaly_type"],
)

print(f"Root cause: {result['final_root_cause']}")
print(f"Confidence: {result['final_confidence']:.0%}")
print(result['final_report'])
```

## Retrieval Layer

### `retrieval.orchestrator.SirenQueryEngine`

Multi-modal retrieval: logs (Pinecone), metrics (TimescaleDB), dependencies (Neo4j), docs (Pinecone).

```python
from retrieval.orchestrator import SirenQueryEngine

engine = SirenQueryEngine()  # Initializes all backends
```

**Methods** (called by agent tools, also available for programmatic use):

```python
# Search logs
logs = engine.query_logs(
    service: str,
    query: str,
    window_start: str,        # ISO timestamp or "now-5m"
    window_end: str,          # ISO timestamp or "now"
    top_k: int = 10,
) -> list[dict]

# Get metrics
metrics = engine.get_metrics(
    service: str,
    window_start: str,
    window_end: str,
) -> dict  # {baseline: {mean, std, p50, p99, ...}, peak: {...}}

# Get service dependencies
deps = engine.get_dependencies(service: str) -> list[str]

# Search documentation
docs = engine.search_docs(query: str, top_k: int = 5) -> list[dict]
```

**Internal:** Resolves ISO/relative timestamps to Unix seconds, applies retrieval weights by (source, incident_type), caches results in Redis (if enabled).

**Example:**
```python
from retrieval.orchestrator import SirenQueryEngine

engine = SirenQueryEngine()

# Find logs about timeout errors
logs = engine.query_logs(
    service="database",
    query="connection timeout",
    window_start="2026-04-21T06:25:00Z",
    window_end="2026-04-21T06:30:00Z",
    top_k=5,
)

for log in logs:
    print(f"[{log['timestamp']}] {log['message']}")
```

## Feedback Layer

### `feedback.store.FeedbackStore`

PostgreSQL persistence for investigations and feedback.

```python
from feedback.store import FeedbackStore

store = FeedbackStore()  # Connects to FEEDBACK_URI
```

**Save investigation:**
```python
store.save_investigation(
    final_state: dict,              # InvestigationState from run_investigation()
    incident_type: str,             # "compute", "database", etc
) -> None
```
Persists to `investigations` table with tool_history, evidence_ledger, final_root_cause, final_confidence.

**Save feedback:**
```python
store.save_feedback(
    incident_id: str,
    verdict: str,                   # "correct", "partial", or "wrong"
    correct_root_cause: str | None = None,
    engineer_notes: str | None = None,
) -> None
```
Persists to `feedback` table; used to compute weights.

**Example:**
```python
store = FeedbackStore()
store.save_investigation(investigation_result, "compute")

store.save_feedback(
    incident_id="compute_20260421T062715",
    verdict="correct",
    engineer_notes="Excellent analysis of connection pool exhaustion"
)
```

### `feedback.optimizer.RetrievalOptimizer`

Computes retrieval weights from feedback verdicts.

```python
from feedback.optimizer import RetrievalOptimizer
from feedback.store import FeedbackStore

store = FeedbackStore()
optimizer = RetrievalOptimizer(store)

weights = optimizer.recompute_weights(min_samples: int = 5)
# Returns: dict[(source, incident_type)] → weight (float, range [0.5, 1.5])
```

Formula: `weight = 0.5 + (correct+1)/(total+2)` (Laplace smoothing)

**Example:**
```python
for (source, incident_type), weight in sorted(weights.items()):
    print(f"{source:20} + {incident_type:10} → {weight:.2f}x")

# Output:
# query_logs           + database     → 1.38x
# query_logs           + compute      → 1.12x
# get_metrics          + compute      → 1.25x
# get_dependencies     + network      → 1.15x
```

### `feedback.stats`

Analytics: accuracy trends, confidence calibration, source effectiveness.

```python
from feedback.stats import (
    compute_accuracy_trend,
    compute_confidence_calibration,
    compute_source_effectiveness,
)
from feedback.store import FeedbackStore

store = FeedbackStore()

# Daily accuracy breakdown
trend = compute_accuracy_trend(store)
# → list[{date: "2026-04-21", accuracy: 0.87, sample_size: 15}]

# Confidence calibration (predicted vs actual)
calib = compute_confidence_calibration(store)
# → list[{confidence: 0.8, actual_accuracy: 0.85, sample_size: 12}]

# Tool usage and success rates
sources = compute_source_effectiveness(store)
# → list[{source: "query_logs", usage_count: 23, success_rate: 0.96}]
```

**Example:**
```python
trend = compute_accuracy_trend(store)
for day in trend[-7:]:
    print(f"{day['date']}: {day['accuracy']:.0%} ({day['sample_size']} investigations)")
```


**Arguments:**
- `metrics_csv` — Path to metrics CSV (required; bundles include logs.csv and traces.csv in same directory)

## Common Workflows

### Workflow: Single Investigation + Feedback

```python
import csv
from detection import detect
from agent.run import run_investigation
from feedback.store import FeedbackStore

# 1. Load metrics from CSV
with open("data/incidents/database_lock/metrics.csv") as f:
    metrics = list(csv.DictReader(f))
    for row in metrics:
        for key in ["error_rate", "latency_p99", "rps", "cpu", "memory"]:
            row[key] = float(row[key])

# 2. Detect anomalies
anomalies, incident = detect(metrics)

# 3. Investigate
result = run_investigation(
    anomalies=anomalies,
    origin_service=incident["affected_services"][0],
    window_start=anomalies[0]["timestamp"],
    window_end=anomalies[-1]["timestamp"],
    incident_id=incident["incident_id"],
    incident_type=incident["anomaly_type"],
    max_steps=15,
)

# 4. Store investigation
store = FeedbackStore()
store.save_investigation(result, incident["anomaly_type"])

print(f"Root cause: {result['final_root_cause']}")
print(f"Confidence: {result['final_confidence']:.0%}")

# 5. (Later) Engineer provides feedback
store.save_feedback(
    incident_id=incident["incident_id"],
    verdict="correct",
    engineer_notes="Accurate diagnosis"
)

# 6. (Weekly) Recompute weights
from feedback.optimizer import RetrievalOptimizer
optimizer = RetrievalOptimizer(store)
weights = optimizer.recompute_weights(min_samples=5)
```

### Workflow: Sensitivity Tuning

```python
from detection import detect

# More sensitive detection (catch smaller anomalies)
anomalies_sensitive, _ = detect(metrics, z_threshold=2.5, contamination=0.1)

# Less sensitive (reduce false positives)
anomalies_strict, _ = detect(metrics, z_threshold=4.0, contamination=0.02)

print(f"Sensitive: {len(anomalies_sensitive)} anomalies")
print(f"Strict: {len(anomalies_strict)} anomalies")
```

### Workflow: Accuracy Analysis

```python
from feedback.store import FeedbackStore
from feedback.stats import compute_accuracy_trend, compute_source_effectiveness

store = FeedbackStore()

# Daily accuracy trend
trend = compute_accuracy_trend(store)
for day in trend[-7:]:
    print(f"{day['date']}: {day['accuracy']:.0%}")

# Which tools are most effective?
sources = compute_source_effectiveness(store)
for tool in sorted(sources, key=lambda x: x["success_rate"], reverse=True):
    print(f"{tool['source']:20} → {tool['success_rate']:.0%} ({tool['usage_count']} uses)")
```

## Troubleshooting

**ImportError: "No module named 'detection'"**
- Run from siren/ root directory
- Verify sys.path includes project root (investigate.py handles this)

**Pinecone: "the $gte operator must be followed by a number"**
- ISO timestamp strings need conversion to Unix seconds
- `resolve_timestamp()` in agent/tools.py and retrieval/orchestrator.py handles this
- Verify PINECONE_API_KEY is valid

**PostgreSQL: "password authentication failed"**
- Check .env: FEEDBACK_URI and TIMESCALE_URI
- Format: `postgresql://user:password@host:port/database`
- Verify credentials match running containers

**Anomalies not detected**
- Try lower z_threshold: `detect(metrics, z_threshold=2.5)`
- Verify all required columns: timestamp, service, error_rate, latency_p99, latency_p50, rps, cpu, memory
- Check metrics have sufficient rows (baseline = first 30%)

**Investigation hangs (>120s)**
- Reduce `max_steps`: `run_investigation(..., max_steps=10)`
- Check Pinecone latency with: `curl -i https://api.pinecone.io/status`
- Enable Redis if available (REDIS_URL in .env)

## Rate Limits

**Anthropic Claude:**
- Default org: 30k tokens/minute
- Investigation: ~30-50k tokens (could hit limit with back-to-back investigations)
- Solution: space out investigations 60s apart, upgrade plan, or use batch mode

**Cohere Reranker:**
- 1000 requests/minute
- Investigation: ~3 rerank calls
- Not a bottleneck at normal volume

**Pinecone:**
- Limits depend on plan
- Free tier: 1M vectors, 2500 queries/month (test plan)
